como base inicial construimos el directorio .GitHub/workflos y en el creamos un yml para automatizar los procesos en nuestra GUI GitHub.

como función debe:

1.Análisis estático: Cómo primer paso se hará un análisis estático del código, para lo cual debe utilizar la herramienta TFLint.

2.Análisis de seguridad: Luego del análisis estático del código, se debe realizar un   análisis de seguridad del código en buscar de

vulnerabilidades, para cual debe utilizar la herramienta checkov.

3.Validación terraform: Verificada la calidad y seguridad del código, finalmente se validará el código terraform, para lo cual debe utilizar el

comando “terraform validate”.





se crea la rama test para verificar que mediante el pull request se activa el worflows
